## KEYWORD ROUTING LOCK — PILOTAGE PAR MOTS-CLÉS

### RÈGLE ABSOLUE
Quand l’utilisateur emploie un mot-clé, l’agent DOIT automatiquement restreindre son raisonnement au sous-système correspondant.

---

## MOT-CLÉ : PONT / BRIDGE
Signifie :
- synchronisation TradingView ? extension ? serveur ? dashboard
- pipeline live
- stabilité du flux
- aucune perte de data
- aucun clignotement

Fichiers prioritaires :
- tradingview-analyzer/content.js
- tradingview-analyzer/background.js
- server.js
- dashboard.html

Interdiction :
- parler design
- parler MT5
- partir sur autre chose

---

## MOT-CLÉ : DASHBOARD
Signifie :
- miroir stable de l’extension
- affichage symbole / timeframe / prix / graphique
- suppression NO DATA parasite
- stabilité d’affichage
- serveur dashboard ONLINE

Fichiers prioritaires :
- dashboard.html
- dashboard.js si existant
- server.js uniquement si la source /extension/data est en cause

Interdiction :
- repartir sur extension si dashboard seul est demandé, sauf preuve d’un maillon cassé avant

---

## MOT-CLÉ : EXTENSION
Signifie :
- popup
- background
- content
- chart-module
- synchronisation avec TradingView

Fichiers prioritaires :
- tradingview-analyzer/popup.js
- tradingview-analyzer/background.js
- tradingview-analyzer/content.js
- tradingview-analyzer/chart-module.js
- tradingview-analyzer/popup.html

Interdiction :
- modifier dashboard si non nécessaire

---

## MOT-CLÉ : ANALYSE
Signifie :
- bouton ANALYSER
- déclenchement agent IA
- lecture marché réel
- retour vocal + texte
- préparation scénario live

Fichiers prioritaires :
- popup.js
- server.js
- agent logique déjà existante

Interdiction :
- toucher design
- inventer un autre scénario

---

## MOT-CLÉ : ENTRER
Signifie :
- confirmation utilisateur prêt
- branchement réel du bouton
- scénario live
- prise de position automatique si conditions validées
- suivi position

Fichiers prioritaires :
- popup.js
- server.js
- logique agent/scénario existante

Interdiction :
- laisser le bouton décoratif

---

## MOT-CLÉ : VOCAL / MUTE
Signifie :
- voix agent
- bouton mute
- texte toujours actif
- voix ON/OFF propre

Fichiers prioritaires :
- popup.js
- popup.html

---

## MOT-CLÉ : STABLE / STABILITÉ
Signifie :
- supprimer clignotement
- supprimer resets inutiles
- supprimer bascules NO DATA
- rendre flux continu

Fichiers prioritaires :
- dashboard.html
- background.js
- server.js

---

## MOT-CLÉ : SUPPRIMER
Signifie :
- retirer proprement un bouton ou bloc inutile
- sans casser design
- sans refonte
- sans créer de vide incohérent

---

## COMPORTEMENT OBLIGATOIRE

Avant de répondre, l’agent doit :
1. détecter le mot-clé principal
2. se limiter au sous-système correspondant
3. éviter la dispersion
4. corriger le problème demandé
5. restart
6. prouver

---

## INTERDICTION ABSOLUE
Si un mot-clé clair est donné :
- ne pas partir sur un autre module
- ne pas faire d’audit global
- ne pas élargir le scope sans preuve
